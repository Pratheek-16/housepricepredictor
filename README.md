# 🏠 House Price Prediction

A machine learning project that predicts house sale prices using regression models, with an interactive Streamlit web application.

---

## 📁 Project Structure

```
model/
├── data.csv               # King County, WA house sales dataset
├── price.ipynb            # Main notebook: EDA, feature engineering, model training
├── app.py                 # Streamlit web application
├── model.pkl              # Saved trained model (generated by notebook)
├── scaler.pkl             # Saved feature scaler (generated by notebook)
├── feature_columns.json   # Feature column order (generated by notebook)
├── city_avg.json          # City average prices (generated by notebook)
├── zip_avg.json           # ZIP code average prices (generated by notebook)
└── README.md              # Project documentation
```

> **Note:** The `.pkl` and `.json` files are auto-generated when you run `price.ipynb`. You only need `data.csv`, `price.ipynb`, and `app.py` to get started.

---

## 📊 Dataset

- **Source:** King County, Washington, USA — House Sales
- **Records:** 4,600 houses
- **Features:** 18 columns including bedrooms, bathrooms, sqft, location, year built, condition, and more
- **Target:** House sale price (min: $7,800 — max: $26,590,000 — avg: $557,906)

### Features Used

| Feature | Description |
|---|---|
| bedrooms | Number of bedrooms |
| bathrooms | Number of bathrooms |
| sqft_living | Interior living area (sqft) |
| sqft_lot | Lot size (sqft) |
| floors | Number of floors |
| waterfront | Waterfront property (0/1) |
| view | View quality (0–4) |
| condition | House condition (1–5) |
| yr_built | Year the house was built |
| city | City name |
| statezip | ZIP code |

### Engineered Features

| Feature | Description |
|---|---|
| house_age | Sale year − year built |
| years_since_reno | Years since last renovation |
| total_sqft | sqft_living + sqft_lot |
| bath_bed_ratio | Bathrooms / bedrooms |
| city_avg_price | Mean price for that city |
| zip_avg_price | Mean price for that ZIP code |

## 🚀 How to Run

### Step 1 — Run the Notebook

Open and run all cells in `price.ipynb`. This will:
- Explore and clean the data
- Engineer features
- Train and compare 4 models
- Save `model.pkl`, `scaler.pkl`, and the JSON files

```bash
jupyter notebook price.ipynb
```

### Step 2 — Launch the Web App

```bash
streamlit run app.py
```

Opens at **http://localhost:8501** in your browser.

## 🖥️ Web App Features

- Sliders for bedrooms, bathrooms, sqft, year built, condition, and view
- Dropdown menus for city and ZIP code (77 ZIP codes available)
- Waterfront and renovation checkboxes
- Live predicted price displayed instantly
- Secondary metrics: price per sqft, city average, house age
- Input feature summary table

---

## 📈 Key Findings

- `sqft_living`, `city_avg_price`, and `zip_avg_price` are the strongest predictors of house price
- Log-transforming the target price significantly improves model performance
- Tree-based models (Random Forest, Gradient Boosting) far outperform linear models on this dataset
- Location encoding (mean price per city/zip) adds strong predictive signal
